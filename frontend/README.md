# MyAgent UI (Vite)

The main chat UI lives in `frontend/` and is built by Vite into `app/templates/dist/`.

## Source of Truth

`frontend/src/` is the source of truth for the UI. Do not hand-edit files under
`app/templates/dist/`; that directory is generated by `npm run build` and will be
overwritten on the next build.

When changing UI behavior or styles:

1. Edit files under `frontend/src/`.
2. Run `npm run build` from `frontend/`.
3. Verify the app using the newly generated `app/templates/dist/` assets.

Only use direct `app/templates/dist/` edits as an emergency hotfix. If that
happens, copy the same change back into `frontend/src/` immediately and rebuild.

## Daily Workflow

Production build:

```bash
cd frontend
npm install
npm run build
```

Development mode:

```bash
cd app
python -m uvicorn webui:fastapi_app --reload --port 8000

cd frontend
npm run dev
```

## Structure

```text
frontend/
|-- index.html              # Page shell
|-- src/
|   |-- main.js             # CSS + path picker + UI entry
|   |-- styles/             # Bundled CSS
|   |-- app/
|   |   |-- index.js        # UI bootstrapper
|   |   |-- modules/        # Feature-split UI modules
|   |   `-- config.js       # Runtime config helpers
|   `-- vendor/
|       `-- myagent_path_picker.js
`-- vite.config.js          # build -> app/templates/dist/
```

## Notes

UI logic is split by feature under `src/app/modules/*`. The bootstrapper currently preserves the old execution order so shared UI state keeps working while the modules are gradually moved toward explicit `import`/`export`.

The backend serves `app/templates/dist/index.html`. If it is missing, the app shows a build prompt instead of falling back to an old template.
